# Метод потрійного резервування (Repetition Code)

Цей проєкт демонструє базові принципи **Forward Error Correction (FEC)** — прямого виправлення помилок. Скрипт симулює роботу передавача та приймача в умовах зашумленого каналу зв'язку, використовуючи код з повторенням та мажоритарну логіку.


## Теоретична база

У реальних телекомунікаційних системах (радіозв'язок, оптоволокно, супутникові канали) сигнал неминуче зазнає деградації через теплові шуми або електромагнітні наведення, що призводить до інверсії окремих бітів. 

Щоб приймач міг самостійно відновити початкові дані без запиту на повторну передачу (ARQ), на стороні передавача в сигнал навмисно вноситься **надмірність (redundancy)**.

### Алгоритм роботи (Triple Modular Redundancy)

1. **Етап кодування (Transmitter):** Кожен біт корисного навантаження (payload) повторюється `N` разів. У даному скрипті використовується потрійне резервування (`N=3`).
   * Логічний `0` кодується як тріада `000`.
   * Логічна `1` кодується як тріада `111`.

2. **Завади в каналі (Channel Noise):** Під час передачі шум може випадково інвертувати один або кілька бітів у тріаді (наприклад, `111` може перетворитися на `101`).

3. **Етап декодування (Receiver):** Приймач розбиває вхідний потік на блоки по 3 біти і застосовує **мажоритарну логіку (Majority Vote)**. Декодер підраховує кількість нулів та одиниць у блоці, приймаючи рішення на користь більшості:
   * `000`, `001`, `010`, `100` $\rightarrow$ Декодується як `0`.
   * `111`, `110`, `101`, `011` $\rightarrow$ Декодується як `1`.


Метод резервування є обчислювально тривіальним, однак вкрай неефективним з точки зору використання спектра. 

* **Пропускна здатність (Code Rate):** Коефіцієнт кодування становить `R = 1/3`. Це означає, що 66.6% переданого трафіку є надлишковим.
* **Межа відновлення:** Цей алгоритм гарантовано виправляє лише $t \leq \lfloor(n-1)/2\rfloor$ помилок. Для `N=3` система здатна виправити максимум **1 помилковий біт** на кожну тріаду. Якщо завада змінить два біти в одному блоці (наприклад, `000` $\rightarrow$ `011`), мажоритарний вентиль видасть хибний результат (`1`).

Для сучасних систем з високими вимогами до пропускної здатності використовуються більш оптимізовані коди з матрицями парності (справжні коди Гемінґа, LDPC, Turbo codes).
