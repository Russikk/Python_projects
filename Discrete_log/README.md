# 🔐 Discrete Logarithm Solver (BSGS Algorithm)

Цей проєкт містить оптимізовану реалізацію алгоритму для розв'язання задачі дискретного логарифма (Discrete Logarithm Problem, DLP) над скінченними полями. 

Задача зводиться до знаходження цілого числа $x$ у модульному рівнянні:
$$g^x \equiv h \pmod p$$

Ця математична проблема лежить в основі криптографічної стійкості асиметричних алгоритмів, таких як обмін ключами **Diffie-Hellman** та електронний підпис **DSA / ECDSA**.

---

##  Алгоритмічна база (Baby-step Giant-step)

Метод повного перебору (Brute Force) має лінійну обчислювальну складність $O(p)$, що робить його абсолютно непридатним для криптографічних модулів великої розрядності.

Даний скрипт реалізує алгоритм Шенкса (Baby-step Giant-step), який використовує архітектурний патерн **Time-Memory Trade-off** (компроміс між часом і пам'яттю). 

### Механіка роботи:
1. **Малі кроки (Baby steps):** Обчислюються та кешуються у хеш-таблицю значення $g^j \pmod p$ для $0 \le j < \lceil\sqrt{p}\rceil$.
2. **Великі кроки (Giant steps):** Обчислюється послідовність $h \cdot (g^{-n})^i \pmod p$.
3. **Пошук колізій:** Кожне значення "великого кроку" перевіряється на наявність у хеш-таблиці "малих кроків". Збіг гарантовано дає шуканий $x$.

### Продуктивність:
* **Часова складність:** $O(\sqrt{p})$
* **Просторова складність (Пам'ять):** $O(\sqrt{p})$
