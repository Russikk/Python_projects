# LFSR-Based Image Scrambling System

Цей проект є реалізацією алгоритму блокового шифрування відео/зображень (Video Scrambling), що базується на псевдовипадкових послідовностях. Система емулює роботу апаратного шифратора для відеопотоку, використовуючи регістр зсуву зі зворотним зв'язком (LFSR).

## Принцип роботи

Метод полягає у **перестановці блоків** зображення. 

1.  **Генерація псевдовипадкової послідовності:**
    Використовується 16-бітний **LFSR (Linear Feedback Shift Register)** з поліномом максимальної довжини:
    $$x^{16} + x^{14} + x^{13} + x^{11} + 1$$
    Цей генератор створює детермінований потік чисел на основі початкового ключа (`SEED`).

2.  **Формування таблиці перестановок:**
    Потік чисел з LFSR використовується в алгоритмі **Fisher-Yates Shuffle** для генерації унікальної таблиці перестановок.

3.  **Обробка зображення:**
    Зображення розбивається на сітку блоків (Grid), які перемішуються згідно з таблицею.

## Структура проекту

* **`common.py`**
    * Містить функцію LFSR генератора. ('lfsr_generator')
    * Реалізує логіку створення таблиці перестановок (`get_permutation_table`).
    * Це спільна бібліотека для шифратора та дешифратора.
    
* **`encrypt_image.py`** — **Модуль шифрування**
    * Зчитує вхідне зображення.
    * Розбиває його на блоки згідно з налаштуваннями сітки.
    * Виконує перестановку блоків.
    * Зберігає результат у файл.

* **`decrypt_image.py`** — **Модуль дешифрування**
    * Зчитує зашифроване зображення.
    * Генерує **обернену** таблицю перестановок, використовуючи той самий ключ (`SEED`).
    * Відновлює початковий порядок блоків.

## Вхідні та вихідні дані

* `input.jpg` — Вихідне зображення.
* `encrypted_image.png` — Зашифроване зображення (результат роботи `encrypt_image.py`).
* `decrypted_image.png` — Відновлене зображення (результат роботи `decrypt_image.py`).

## Configuration

* **`SEED_KEY`** (наприклад, `44257`): 16-бітний ключ шифрування. **Має співпадати** у шифратора та дешифратора.
* **`GRID`** (наприклад, `(40, 30)`): Параметр розбиття зображення `(рядки, стовпці)`. 
    * Збільшення значень робить блоки меншими, а шифрування — надійнішим
